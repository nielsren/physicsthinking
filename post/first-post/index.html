GithubPages教程

December 10, 2025

这是我用 Hugo 搭建的第三篇博客！

Hugo 网站更新与发布完整教程（PowerShell + 完整路径版）

第一步：打开 PowerShell

方法一（推荐）:

  1. 在 Hugo 项目文件夹任意位置按住 Shift + 右键
  2. 选择“在此处打开 PowerShell 窗口”

方法二:

  1. 按 Win + R
  2. 输入 powershell 回车

第二步：进入 Hugo 网站根目录

假设 Hugo 项目路径为： D:\HugoSites\myblog

PowerShell 输入:

cd D:\HugoSites\myblog

验证是否进入:

dir

应看到:

content
themes
layouts
static
config.toml
public

第三步：创建新的 Markdown 文章文件

Hugo 命令创建（推荐）:

hugo new post/first-post.md

文章路径:

D:\HugoSites\myblog\content\post\first-post.md

编辑文章内容（示例）:

---
title: "我的第一篇 Hugo 文章"
date: 2025-12-10
draft: false
---

这是我的第一篇博客文章。

注意: draft 必须设为 false 否则不会生成到 public。

第四步：生成 public 静态网站

cd D:\HugoSites\myblog
hugo

第五步：进入发布目录 public（完整路径）

cd D:\HugoSites\myblog\public

第六步：提交到 GitHub Pages（完整命令）

# 添加所有修改或新文件
git add -A

# 提交更新
git commit -m "update site content"

# 推送到远程仓库 main 分支
git push origin main

如果使用 gh-pages 分支，把最后一行改成 git push origin gh-pages

第七步：查看网站

浏览器访问:

https://你的GitHub用户名.github.io/仓库名

最终完整操作流程

cd D:\HugoSites\myblog
hugo new post/文章名.md
# 编辑文章后保存
hugo
cd D:\HugoSites\myblog\public
git add -A
git commit -m "update site content"
git push origin main

注意事项

  1. 每次修改内容必须先执行 hugo
  2. git push 必须在 public 目录
  3. draft 必须为 false

可选优化：一键发布 bat 文件

创建 deploy.bat 文件，内容如下:

@echo off
cd D:\HugoSites\myblog
hugo
cd D:\HugoSites\myblog\public
git add -A
git commit -m "update site content"
git push origin main
pause

双击 deploy.bat 即可一键生成文章并发布网站。
